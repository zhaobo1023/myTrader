项目结构如下：
[粘贴上面的目录结构]

我已有的基础：
- mysql数据库，已有A股日线行情表和因子表
- 已验证有效因子：mom_20, mom_60, price_vol_diverge, net_profit_growth, revenue_growth

### 分任务Prompt（按顺序执行）

**Task 1：外部数据采集模块**

```
基于已有的data/fetcher.py，新增外部宏观数据拉取功能。

需要实现：
1. 用AkShare拉取以下数据，存入quant.db的macro_data表：
   - WTI原油价格（日频）
   - 黄金价格（日频）  
   - 美元指数（日频）
   - VIX恐慌指数（日频）
   - A股北向资金净流入（日频）

2. macro_data表结构：(date TEXT, indicator TEXT, value REAL, PRIMARY KEY(date, indicator))

3. 支持增量更新：只拉取数据库中最新日期之后的数据

4. 每个数据源单独try-catch，一个失败不影响其他的

请先输出AkShare中每个数据对应的正确函数名（先验证接口存在），再写实现代码。
```

```
在factors/macro.py中实现宏观因子计算。

输入：从quant.db读取macro_data表
输出：因子值DataFrame，格式与现有因子表一致(date, indicator, value)

需要实现的因子：
1. 原油20日涨跌幅（oil_mom_20）
2. 黄金20日涨跌幅（gold_mom_20）
3. 美元指数20日变化（dxy_mom_20）
4. VIX 5日均值（vix_ma5）
5. 北向资金5日累计净流入（north_flow_5d）

计算完成后追加写入quant.db的macro_factors表。
```

**Task 3：ETF因子测试模块**

```
实现research/etf_test.py，测试宏观因子对ETF的预测力。

功能：
输入：ETF代码列表、因子名称列表、测试区间
输出：每个(ETF, 因子)组合的IC值、ICIR、滚动IC图

ETF标的：
- 159930（能源化工）
- 515220（煤炭）
- 518880（黄金）
- 513130（恒生科技）

实现逻辑：
1. 从quant.db读取ETF日线价格（如果没有先从AkShare拉取存库）
2. 计算ETF未来N日收益（N=5,10,20）
3. 与宏观因子做时序相关性（不是截面IC，是时序pearson相关）
4. 输出相关性矩阵热力图 + 滚动相关性折线图

注意：这里是时序IC（单标的），不是截面IC（多股票），计算方式不同，请用时序pearson相关系数。
```

**Task 4：滚动IC监控模块**

```
实现monitor/rolling_ic.py，这是每日自动化的核心监控。

功能：
1. 对factors表中所有因子，计算60日滚动IC
2. 判断因子状态：
   - 有效：近60日IC绝对值均值 > 0.03 且 ICIR绝对值 > 0.4
   - 衰减：近60日IC绝对值均值 在 0.01-0.03 之间
   - 失效：近60日IC绝对值均值 < 0.01
3. 结果存入factor_status表：(date, factor_name, rolling_ic, rolling_icir, status)
4. 状态发生变化时（有效→衰减，衰减→失效）记录到factor_alerts表

输出：今日因子状态汇总字典，供reporter.py调用。
```

------

**Task 5：每日自动化主流程**

```
实现daily_run.py，整合所有模块的每日执行逻辑。

执行顺序：
1. 检查基础数据都存储成功（有另外的定时任务负责任务写入)
2. 计算技术/基本面/宏观因子 → 追加因子库
3. 运行滚动IC监控 → 更新factor_status
4. 如有因子状态变化 → 触发邮箱推送推送 163邮箱

要求：
- 每步执行前打印进度日志
- 任何一步失败只记录错误，继续执行后续步骤
- 最后输出今日执行摘要：成功几步/失败几步/因子状态变化
- 支持命令行参数：--date 指定日期（默认昨天），--full 触发全量重算
```

------

**Task 6：多因子组合研究**

```
实现research/combination.py，研究多因子组合效果。

分两步：

第一步：因子去冗余
- 计算所有有效因子的相关性矩阵
- 相关性绝对值 > 0.7 的因子对，保留ICIR更高的那个
- 输出去冗余后的因子列表

第二步：组合合成
实现两种合成方式：
1. 等权合成：所有因子标准化后等权相加
2. IC加权合成：用历史IC均值作为权重

用Alphalens对合成因子跑完整验证报告，和单因子对比：
- 合成因子IC是否优于最好的单因子
- 分层收益是否更单调
- 换手率是否可接受（过高说明组合不稳定）
```

### 使用方式

每次开启Claude Code会话，先粘贴**主Prompt**建立上下文，然后粘贴对应的**分任务Prompt**。

按Task 1→2→3→4→5→6顺序做，因为后面的任务依赖前面的模块。Task 4和Task 5是核心框架，Task 3和Task 6是研究模块，可以并行。

每个Task做完测试跑通后再开下一个，不要一次性全部丢给它。
