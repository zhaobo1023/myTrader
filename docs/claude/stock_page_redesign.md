# /stock 个股分析页面改造方案

> 日期：2026-04-27
> 状态：方案设计（待评审）

---

## 一、现状分析

### 1.1 当前页面结构

**`/stock` 页面** — 7 个 Tab：
| Tab | 功能 | 数据来源 |
|-----|------|----------|
| K线图 | 日/周/月 K 线 + MA/BOLL/MACD/RSI/KDJ | `/api/chart/combined` |
| 一页纸研究 | SSE 流式深度研报（A-I 九节） | `/api/analysis/one-pager/generate` |
| 综合分析 | 五步法分析（估值/盈利/成长/资本/技术） | `/api/analysis/comprehensive/generate` |
| 技术面分析 | 规则化技术信号打分 + HTML 报告 | `/api/analysis/reports/by-stock` |
| 基本面分析 | 同综合分析（五步法） | 同上，type=fundamental |
| 个股动态 | 7 天新闻 + LLM 情感分析 | `/api/analysis/stock-news` |
| 风控情况 | PE/PB 5 年历史 + 百分位 | `/api/analysis/valuation/stock/{code}/history` |

**`/analysis` 页面** — 4 个 Tab：
| Tab | 功能 | 与 /stock 重叠度 |
|-----|------|-----------------|
| 技术面报告 | 同 /stock 技术面分析 | 完全重复 |
| 一页纸深度研究 | 同 /stock 一页纸研究 | 完全重复 |
| 综合研报 | 同 /stock 综合分析 | 完全重复 |
| 估值历史 | 同 /stock 风控情况 | 高度重复 |

### 1.2 发现的问题

**功能问题：**
1. ~~周/月 K 线指标无法匹配~~ -- 已修复（chart_service.py SQL 重写）
2. ~~`trade_technical_indicator` 表为空~~ -- 已修复 NaN 转换 bug，回填中
3. BOLL/KDJ 等指标显示"近5日数据不足" -- 根因是表空，回填后可解决

**架构问题：**
1. `/stock` 和 `/analysis` 页面 **4 个功能完全重复**，用户不清楚该用哪个
2. `/analysis` 页面默认展示"最近报告网格"，与个股分析的定位冲突
3. 报告生成有 3 种方式（同步/SSE/Celery 异步），代码路径分散
4. Markdown 渲染在两个页面用不同实现（`marked` vs 内联 `renderMarkdown()`）
5. 股票搜索组件在两个页面独立实现

**用户体验问题：**
1. Tab 过多（7 个），认知负担大
2. "综合分析"和"基本面分析"本质上是同一个五步法分析器，区分不清
3. 没有"快速概览"入口，每次都要手动选 Tab 逐项查看
4. 报告生成配额只在报错时才显示
5. 估值数据散布在"风控"和"综合分析"两处

## 二、目标用户场景

以投资经理日常工作流为锚点：

### 场景 A：晨会快速过股
> 打开一只股票，30 秒内了解：当前价位/趋势/估值分位/最近信号

**需要：** K 线 + 关键指标摘要 + 估值分位 -- 一屏搞定

### 场景 B：深度研究决策
> 准备买入/卖出，需要全面分析

**需要：** 一页纸研报 / 综合五步法 -- 流式生成，等待时能看 K 线

### 场景 C：持仓监控
> 定期检查持仓股的技术面变化和新闻

**需要：** 技术信号预警 + 新闻动态 -- 不用生成新报告，看已有的即可

### 场景 D：行业比较
> 同行业几只股票做横向对比

**需要：** 估值对比 + 技术面对比 -- 当前缺失，后续规划

## 三、改造方案

### 3.1 核心思路：合并为统一的 `/stock` 页面

**废弃 `/analysis` 页面**，将其独有功能（报告网格浏览）合并到其他入口。
`/stock` 成为个股分析的**唯一入口**，Tab 从 7 个精简为 4 个。

### 3.2 新 Tab 结构

```
/stock?code=600519.SH
  |
  +-- [概览]        晨会快速过股（新增）
  +-- [K线与技术面]  K线图 + 技术信号列表（合并原"K线图"+"技术面分析"）
  +-- [深度研报]     一页纸 + 综合分析（合并原3个Tab）
  +-- [动态与风控]   新闻 + 估值历史（合并原"个股动态"+"风控情况"）
```

### 3.3 各 Tab 详细设计

#### Tab 1: 概览（新增）

**定位：** 30 秒快速了解一只股票的全貌

**布局（上下三栏）：**
```
+---------------------------------------------------------------+
| [价格区]  600519.SH 贵州茅台                                    |
| 现价 1,680.00  涨跌 +2.3%  成交额 42亿  换手 0.8%              |
+---------------------------------------------------------------+
| [技术摘要]                     | [估值摘要]                     |
| MA趋势: 多头排列 [OK]         | PE-TTM: 28.3 (历史50分位)     |
| MACD: 金叉3日 [OK]            | PB: 10.2 (历史75分位)         |
| RSI(14): 62 [OK]              | 近1月涨幅: +5.2%              |
| KDJ: K=75 D=68 J=89 [OK]     | 近1年涨幅: -8.3%              |
| BOLL: 价格在中轨上方 [OK]     | 所属行业: 食品饮料              |
+---------------------------------------------------------------+
| [最近信号]                                                      |
| 04-25 技术面扫描: 综合82分 [OK] MA多头排列, MACD金叉            |
| 04-24 个股动态: 一季报净利同比+15%, 机构增持 (+2.3%)            |
+---------------------------------------------------------------+
```

**数据来源：**
- 价格区：复用 `/api/market/stock-detail/{code}` 或 K 线最后一条数据
- 技术摘要：`/api/chart/combined` 最新一条指标数据 + 解读逻辑（复用 KLineChart 的解读函数）
- 估值摘要：`/api/analysis/valuation/stock/{code}/history` 取最新值
- 最近信号：`/api/analysis/reports/by-stock` 取最近 3 天

**实现复杂度：** 中等。需要新建 `StockOverview` 组件，数据接口已有。

#### Tab 2: K线与技术面（合并）

**定位：** 图表分析 + 历史技术信号

**布局：**
```
+---------------------------------------------------------------+
| [K线图区 - 占 70% 高度]                                        |
| 日K / 周K / 月K    指标: MA BOLL VOL MACD RSI KDJ             |
| (复用现有 KLineChart 组件，无需改动)                            |
+---------------------------------------------------------------+
| [技术信号列表 - 占 30% 高度，可折叠]                            |
| 最近 3 天技术面扫描报告                                         |
| > 04-25 综合82分 [展开可看信号详情]                             |
| > 04-24 综合78分                                                |
+---------------------------------------------------------------+
```

**变更点：**
- 将原"技术面分析" Tab 的报告列表移到 K 线图下方
- 报告卡片默认折叠，只显示日期+分数+等级

#### Tab 3: 深度研报（合并）

**定位：** AI 生成的深度分析报告

**布局：**
```
+---------------------------------------------------------------+
| 报告类型: [一页纸研究] [综合五步法]     [生成报告]              |
+---------------------------------------------------------------+
| [报告内容区]                                                    |
| (已有缓存) 直接展示 Markdown                                   |
| (无缓存)   点击"生成报告"后 SSE 流式展示                       |
+---------------------------------------------------------------+
| [历史报告]  展开可查看过去的报告                                |
+---------------------------------------------------------------+
```

**变更点：**
- 合并原"一页纸研究"+"综合分析"+"基本面分析"为一个 Tab
- 报告类型用按钮组切换（不是子 Tab）
- 去掉"基本面分析"独立 Tab -- 综合五步法已包含基本面
- 统一使用共享的 `MarkdownViewer` 组件

#### Tab 4: 动态与风控（合并）

**定位：** 新闻追踪 + 估值监控

**布局：**
```
+---------------------------------------------------------------+
| [估值概览 - 紧凑版]                                            |
| PE-TTM: 28.3 (p50)  PB: 10.2 (p75)                           |
| [展开] 查看 5 年历史图表                                       |
+---------------------------------------------------------------+
| [新闻列表]                                                     |
| 近 7 天新闻 + 事件分类标签                                     |
| [LLM 情感分析] 按钮触发                                        |
+---------------------------------------------------------------+
```

**变更点：**
- 合并原"个股动态"+"风控情况"
- 估值图表默认收起，只显示当前值+分位

### 3.4 `/analysis` 页面处理

**方案：** 保留但改为"报告浏览器"定位

- 去掉股票搜索和 Tab 切换功能
- 只保留"最近报告网格"视图
- 报告卡片点击后跳转到 `/stock?code=xxx&tab=report`
- 导航菜单中降级为二级入口（或放到"更多"下拉菜单中）

### 3.5 共享组件提取

| 组件 | 来源 | 复用位置 |
|------|------|----------|
| `StockSearchBox` | 统一搜索组件 | /stock, /analysis, /portfolio |
| `MarkdownViewer` | 统一 Markdown 渲染 | 所有报告展示 |
| `TechReportCard` | 技术报告卡片 | /stock Tab2, /analysis 网格 |
| `ValuationMini` | 估值摘要组件 | /stock Tab1, Tab4 |
| `useReportTask` | 报告生成 Hook | /stock Tab3 |
