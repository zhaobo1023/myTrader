## 因子对比分析

### 2025年因子表现对比

| 因子 | 总交易数 | 胜率 | 总收益 | 年化收益 | 最大回撤 | 夏普比率 | 排名 |

### 因子特点分析

#### pure_mv（纯市值因子）✨
- **优势**：胜率最高（46.28%），夏普比率最好（-0.89）
- **劣势**：回撤较大（-34.13%）
- **适用场景**：市场风格切换期，小市值弹性
- **风险**：流动性风险高，易受市场情绪影响

#### PEG（市盈率增长比）
- **优势**：回撤控制最好（-29.82%），总收益相对较高
- **劣势**：胜率一般（43.83%）
- **适用场景**：考虑基本面质量的成长股
- **风险**：依赖EPS增速数据，可能存在滞后

#### PE（市盈率）
- **优势**：简单直观，数据易得
- **劣势**：表现最差（收益-39.38%，夏普-2.39）
- **适用场景**：价值投资策略
- **风险**：微盘股低PE可能是价值陷阱

## 问题诊断与修复

### 🔴 问题1：因子计算返回空结果

#### 症状
```
Factors returned: 0
Columns: ['stock_code', 'peg']
No stocks found for 2025-01-02
```

#### 根本原因
`factors_cached.py` 中日期类型不匹配：
- 数据库返回：`datetime.date` 对象
- 查询条件：字符串 `'2025-01-02'`
- 结果：类型不匹配，查询返回空DataFrame

#### 修复方案
```python
# factors_cached.py line 56-59
self.df_daily_basic['trade_date'] = pd.to_datetime(
    self.df_daily_basic['trade_date']
).dt.strftime('%Y-%m-%d')

self.df_daily['trade_date'] = pd.to_datetime(
    self.df_daily['trade_date']
).dt.strftime('%Y-%m-%d')
```

#### 修复验证
```python
# 修复前
2025-01-02 的数据行数: 0

# 修复后
2025-01-02 的数据行数: 5091
Factors returned: 740
有效PEG数量: 286
```

### 🔴 问题2：交易配对逻辑错误

#### 症状
```
平均持仓天数: -26.3  # 负数！
```

#### 根本原因
`metrics.py` 中买卖交易配对逻辑错误：
- 每次都用 `buy.iloc[-1]`（最后一个买入）
- 导致同一个买入被多次配对
- 持仓天数计算错误（卖出日期 < 买入日期）

#### 修复方案
```python
# 修复前
buy = buy_trades[buy_trades['stock_code'] == sell['stock_code']]
buy = buy.iloc[-1]  # ❌ 总是取最后一个

# 修复后
buy_queue = {}  # {stock_code: [buy_indices]}
# 使用队列先进先出
buy_idx = buy_queue[code].pop(0)
buy = buy_trades.iloc[buy_idx]
```

#### 修复验证
```python
# 修复前
平均持仓天数: -26.3

# 修复后
平均持仓天数: 21.4 ✅
```

### D. 关键指标说明

#### 夏普比率 (Sharpe Ratio)
- **定义**：(年化收益 - 无风险利率) / 波动率
- **意义**：每承担一单位风险获得的超额收益
- **标准**：>1优秀，0.5-1良好，<0.5不佳
- **当前**：-1.12（不理想）

#### 最大回撤 (Max Drawdown)
- **定义**：从最高点到最低点的跌幅
- **意义**：最坏情况下的损失
- **标准**：<10%优秀，10-20%良好，>20%需警惕
- **当前**：-29.82%（风险较高）

#### 胜率 (Win Rate)
- **定义**：盈利交易占总交易的比例
- **意义**：策略盈利的稳定性
- **标准**：>60%优秀，50-60%良好，<50%不佳
- **当前**：43.83%（不理想）

   - 建议调整至5-10天
