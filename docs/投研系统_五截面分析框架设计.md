# 投研系统 · 五截面分析框架设计方案

> 基于现有技术面+资金面，整合基本面（估值）、情绪面、资本周期面，构建对公司/行业当前状态的完整截面刻画  
> 文档版本：v1.0 · 2026-04-05

---

## 一、两份参考报告的亮点与不足

### 1.1 宁德时代估值分析报告

**亮点**

- **多方法并列**：8种估值方法同时呈现（清算→重置→利润→FCF→DCF→天花板），每种方法都标注"vs当前市值"的偏差百分比，对比感极强
- **天花板矩阵**：市占率×净利率×PE的27种组合全部展开，是判断"乐观情景到底乐观多少"的最有力工具
- **Gordon模型反推**：从当前PE反推市场隐含的永续增长率（7.3%），这个视角极少见，能直接检验定价是否合理
- **三情景标准化**：悲观/中性/乐观在FCF和利润估值上都有明确参数，可复现

**不足**

- **业务利润口径不透明**：报表利润¥722亿 vs 业务利润¥474亿，差了248亿，调整项是什么没有说明，读者无法验证
- **合理PE取25x缺乏锚定**：为什么是25x？应该接历史分位数引擎（如40%分位对应30x），不应凭空给定
- **各方法结论散但没有综合判断**：¥1.2万亿～¥3.9万亿的区间极大，报告没有给出综合中位估值，读者自己算
- **估值与资本周期完全割裂**：估值低不等于可以买，需要结合股东行为和阶段判断，单独看估值是不完整的
- **DCF参数假设偏激进**：一阶段22%增长5年+二阶段10%增长5年，在关税风险加剧背景下可信度存疑，缺乏敏感性分析

**借鉴到我们系统**

- 多方法并列结构直接采用，但合理PE一律接历史分位数引擎输出，不允许硬编码
- 天花板矩阵作为"乐观情景上限"标配模块
- 新增：将估值综合结论与资本周期阶段绑定（阶段②低估=强买；阶段④低估=陷阱）

### 1.2 恒生指数情绪分析报告

**亮点**

- **核心事件跟踪表**：时间/事件/方向/力度/判定标准五列，是公告NLP Pipeline的直接输出格式，结构极优
- **三重交易逻辑叙述**：把当前市场的多个并行叙事（AI主线/HALO策略/业绩切换）说清楚，不是单一逻辑，符合真实市场复杂性
- **情绪-估值联动段落**：把情绪高分和PE历史分位数打通分析，这是少见的跨截面联动设计，我们要标准化这个做法
- **末尾跟踪清单**：资金/情绪/基本面/地缘/政策五类指标分列，可以直接转化为定期刷新的数据任务清单

**不足**

- **维度权重无标定**：35%/25%/20%/15%/5%的设定没有历史回测支撑，无法验证哪个权重组合预测力最强
- **分数没有历史分布参考**：综合情绪55分是高还是低？需要历史均值和标准差才有意义（类似PE历史分位数的做法）
- **指数报告不下钻成分股**：情绪分析只到指数层，无法知道哪些成分股情绪最极端，错过结构性机会
- **适用性局限**：这套框架直接套用指数，用到个股时需要调整——个股没有"南向资金"，需要替换为机构持仓变化
- **无历史情绪回溯**：情绪评分序列化之后，才能做"历史上这个分位通常对应什么后续涨跌"的统计检验

**借鉴到我们系统**

- 事件跟踪表作为所有公司情绪面的标准数据结构，字段：时间/事件/方向/力度/信号类别/已兑现?
- 情绪评分做历史序列化存储，每次更新记一条，积累后可做情绪均值回归分析
- 指数情绪报告作为行业情绪代理，每个行业指定一个代表性ETF/指数做情绪跟踪

## 二、五截面框架总览

### 2.1 截面定义与数据来源

| 截面 | 核心问题 | 状态 | 主要数据源 |
|------|---------|------|-----------|
| **技术面** | 价格结构说什么？ | ✅ 已实现 | 行情数据，已有因子库 |
| **资金面** | 钱在往哪里流？ | ✅ 已实现 | ETF流量，北向/南向资金，龙虎榜 |
| **基本面** | 公司本身质地怎样？ | 🔧 待完善 | 财报，估值历史分位，一致预期 |
| **情绪面** | 市场情绪在什么位置？ | 🔧 待构建 | 事件跟踪表，NLP公告，舆情 |
| **资本周期面** | 产业资本怎么投票？ | 🔧 待构建 | 公告NLP，股东行为，供给信号 |

### 2.2 综合评分权重（建议初始值，后续用IC回测校准）

```
综合评分 = 技术面×15% + 资金面×20% + 基本面×30% + 情绪面×15% + 资本周期×20%
```

权重逻辑：基本面决定价值锚（权重最高）；资本周期决定大方向；资金面是短期催化；技术面和情绪面是择时辅助。

## 三、各截面详细设计

### 3.1 基本面截面（对标估值报告）

**评分逻辑（0～100分）**

```
基本面评分 = 盈利质量分(40%) + 估值吸引力分(40%) + 成长确定性分(20%)
```

**盈利质量子项**
- ROE水平及趋势：ROE>20%且改善 → +25；ROE下滑 → 扣分
- 现金流质量：经营现金流/净利润 > 1 → +10；<0.5 → -10
- 资产负债率：行业调整后

**估值吸引力子项**
- PE历史分位（5年）：<20% → +30；>80% → -30；线性映射
- PB历史分位（5年）：同上逻辑，权重减半
- FCF收益率：>6% → 加分

**成长确定性子项**
- 管理层指引可信度（历史指引vs实际达成率）
- 一致预期增速：取中位数，<10% → 低分；>30% → 高分

**产出字段**

```json
{
  "fundamental_score": 78,
  "pe_current": 26.2,
  "pe_quantile_5yr": 0.22,
  "pe_label": "低估",
  "valuation_methods": [
    {"method": "报表利润", "amount": "1.8万亿", "vs_market": -0.045},
    {"method": "DCF三阶段", "amount": "3.9万亿", "vs_market": 1.083}
  ],
  "gordon_implied_g": 0.073,
  "expected_return_2yr": 0.884,
  "confidence": "中"
}
```

**自动化程度**：PE/PB分位数 → 全自动；多方法估值 → 规则计算；文字解读 → LLM生成

### 3.2 情绪面截面（对标情绪报告）

**评分维度（对应不同标的类型调整权重）**

| 维度 | 个股权重 | 指数权重 | 数据来源 |
|------|---------|---------|---------|
| 资金面情绪 | 30% | 35% | ETF净流入/机构持仓变化 |
| 量价情绪 | 25% | 25% | 成交量/换手率/量价背离 |
| 市场共识情绪 | 20% | 20% | 分析师评级变化/情绪词频 |
| 板块联动情绪 | 20% | 15% | 行业相关度/板块轮动位置 |
| 地缘/宏观情绪 | 5% | 5% | 新闻事件NLP |

**评分历史化**：每次计算的情绪分写入时序表，积累≥60个样本后，输出"历史分位"（如：当前51分，历史62%分位）

**事件跟踪表结构**（核心，直接复用情绪报告格式）

```sql
CREATE TABLE sentiment_events (
  id BIGINT PRIMARY KEY,
  code VARCHAR(20),          -- 标的代码，指数用指数代码
  event_date DATE,
  event_text VARCHAR(300),   -- 事件描述
  direction ENUM('正面','负面','中性'),
  magnitude ENUM('高','中','低'),
  category ENUM('资金','业绩','政策','地缘','产业','技术'),
  is_verified BOOLEAN,       -- 是否已兑现
  verified_result VARCHAR(100),
  source VARCHAR(100)
);
```

**自动化程度**：量价计算 → 全自动；事件抓取 → 爬虫+NLP半自动；最终分值综合 → 规则引擎

## 四、公司深度跟踪系统

### 4.1 跟踪池设计（支持50家公司）

**分层管理**

```
深度跟踪（≤10家）
  · 全五截面，每周完整刷新
  · 每家公司维护一个"当前交易逻辑摘要"（≤200字，人工维护或AI辅助）
  · 重要事件发生时实时推送

标准跟踪（≤30家）
  · 五截面评分每周更新，但文字报告每月生成一次
  · 评分变化超过阈值（±15分）时触发深度报告

关注观察（≤50家）
  · 仅技术面+资金面实时更新
  · 基本面季报季更新
  · 触发进入"标准跟踪"的条件：资本周期阶段变化 or 大股东重要公告
```

**公司画像文件（每家公司维护一份YAML配置）**

## 五、数据库新增表设计

```sql
-- 基本面快照（每周一条）
CREATE TABLE fundamental_snapshots (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  code VARCHAR(10),
  snap_date DATE,
  fundamental_score TINYINT,
  pe_ttm DECIMAL(8,2),
  pe_quantile_5yr DECIMAL(5,3),
  pb DECIMAL(6,2),
  pb_quantile_5yr DECIMAL(5,3),
  roe DECIMAL(6,4),
  revenue_yoy DECIMAL(6,4),
  profit_yoy DECIMAL(6,4),
  fcf DECIMAL(12,2),
  net_cash DECIMAL(12,2),
  expected_return_2yr DECIMAL(6,4),
  valuation_json JSON,           -- 多方法估值详情
  INDEX idx_code_date (code, snap_date)
);

-- 情绪面评分序列
CREATE TABLE sentiment_scores (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  code VARCHAR(20),              -- 股票或指数代码
  score_date DATE,
  composite_score TINYINT,       -- 综合0~100
  score_fund TINYINT,            -- 资金面情绪
  score_price_vol TINYINT,       -- 量价情绪
  score_consensus TINYINT,       -- 市场共识
  score_sector TINYINT,          -- 板块联动
  score_macro TINYINT,           -- 地缘/宏观
  historical_quantile DECIMAL(4,3),  -- 当前分在历史分布的位置
  label VARCHAR(20),             -- 如"中性偏多"
  INDEX idx_code_date (code, score_date)
);

-- 情绪事件跟踪
CREATE TABLE sentiment_events (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  code VARCHAR(20),
  event_date DATE,
  event_text VARCHAR(300),
  direction ENUM('正面','负面','中性'),
  magnitude ENUM('高','中','低'),
  category ENUM('资金','业绩','政策','地缘','产业','技术','股东'),
  is_verified BOOLEAN DEFAULT FALSE,
  verified_result VARCHAR(200),
  source VARCHAR(100),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_code (code),
  INDEX idx_date (event_date)
);

-- 综合评分历史（五截面汇总）
CREATE TABLE composite_scores (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  code VARCHAR(20),
  score_date DATE,
  score_technical TINYINT,
  score_fund_flow TINYINT,
  score_fundamental TINYINT,
  score_sentiment TINYINT,
  score_capital_cycle TINYINT,
  composite_score TINYINT,       -- 加权综合
  direction ENUM('强多','偏多','中性','偏空','强空'),
  phase TINYINT,                 -- 资本周期阶段1~5
  key_signal TEXT,               -- AI生成的核心信号摘要
  INDEX idx_code_date (code, score_date)
);

-- 跟踪公司池
CREATE TABLE watchlist (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  code VARCHAR(20),
  name VARCHAR(100),
  tier ENUM('deep','standard','watch') DEFAULT 'watch',
  industry VARCHAR(50),
  added_date DATE,
  profile_yaml TEXT,             -- 公司画像YAML全文
  current_thesis TEXT,           -- 当前交易逻辑摘要（≤200字）
  thesis_updated_at DATE,
  is_active BOOLEAN DEFAULT TRUE,
  UNIQUE KEY uk_code (code)
);
```

## 八、差异化竞争优势总结

| 维度 | 市面上免费研报 | 我们的系统 |
|------|-------------|-----------|
| 估值方法 | 单一PE/PB | 8种方法并列+情景矩阵 |
| 情绪量化 | 定性描述 | 历史分位数+5维评分序列 |
| 股东行为 | 事后披露 | 公告NLP实时解析+信号权重 |
| 截面整合 | 各自独立 | 五截面加权综合+联动规则 |
| 个性化 | 无 | 每家公司独立画像+个性化预警 |
| 持续跟踪 | 不连续 | 日度/周度/事件触发分层刷新 |
| 可操作性 | 泛泛结论 | 明确配置建议+阶段转换触发条件 |
