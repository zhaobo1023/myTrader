## 二、数据采集层（最关键）

### 2.1 模块① — 公告NLP Pipeline（难度最高，价值最大）

这是与市场上普通报告拉开差距的核心，股东行为数据靠这里产出。

**数据源优先级：**

| 来源 | 覆盖范围 | 获取方式 | 成本 |
|------|---------|---------|------|
| 巨潮资讯 cninfo.com.cn | 全量A股公告 | HTTP爬虫 + PDF解析 | 免费 |
| 上交所/深交所公告API | 官方原始数据 | REST接口 | 免费 |
| akshare `stock_notice_report` | 公告摘要 | Python接口 | 免费 |
| tushare `anns` | 结构化公告 | API（积分限制）| 低成本 |
| Wind公告库 | 最完整 | SDK | 贵 |

**Pipeline流程：**
```
巨潮爬虫抓取PDF/HTML
    → 分类模型（5类：减持/回购/再融资/员持/增持）
    → 实体提取LLM（结构化JSON输出）
    → 信号打分（正面/负面/权重）
    → 写入 MySQL announcements 表
```

**实体提取Prompt（核心）：**
```
你是A股公告信息提取专家。从以下公告文本中提取结构化信息，
只输出JSON，不输出其他内容：

{
  "type": "减持|回购|再融资|员持|增持",
  "subject": "操作主体名称",
  "subject_role": "控股股东|创始人|国家队|产业资本|公司本身",
  "amount_shares": 数量（万股）,
  "amount_rmb": 金额（亿元）,
  "price_avg": 均价,
  "pct_of_total": 占总股本%,
  "date_start": "YYYY-MM-DD",
  "date_end": "YYYY-MM-DD",
  "sentiment": "正面|负面|中性",
  "weight": "高|中|低",  // 基于主体身份和规模
  "raw_key_sentence": "原文关键句（<30字）"
}

公告文本：
{{announcement_text}}
```

**信号权重规则：**
```python
WEIGHT_RULES = {
    ("减持", "创始人", "高"): -3,      # 如黄世霖 → 强负面
    ("减持", "国家队", "高"): -2,       # 汇金减持 → 中负面（逆周期转向）
    ("减持", "产业资本", "中"): -2,
    ("回购", "公司本身", "高"): +2,
    ("员持", "公司本身", "高"): +2,     # 深折价员持 → 强正面
    ("增持", "国家队", "高"): +3,       # 国家队增持 → 强正面
}
```

## 三、信号计算层（规则引擎）

### 3.1 五维评分引擎

每个维度1~5分，规则打分为主，LLM做边界复核。

```python
class FiveDimensionScorer:
    
    def score_earnings_quality(self, roe_trend, cashflow_quality) -> int:
        """
        1: ROE持续改善，现金流优秀
        3: ROE高位走弱或现金流一般  
        5: 连续亏损或ROE极低
        """
    
    def score_supply_signal(self, capex_yoy, ipo_pace, pe_ratio_industry) -> int:
        """
        1: CAPEX大幅收缩，IPO停发
        3: CAPEX平稳
        5: CAPEX暴增，IPO扎堆破发
        """

        """IPO数量 + 破发率"""
    
    def score_shareholder_return(self, announcement_signals: list) -> int:
        """
        从公告NLP输出的signals列表计算
        正面信号多 → 1~2，负面信号多 → 4~5
        """
        net_score = sum(s['weight_score'] for s in announcement_signals)
        # 映射到1~5
    
    def score_valuation_sentiment(self, pe_quantile_5yr) -> int:
        """
        0~20% → 1, 20~40% → 2, 40~60% → 3,
```

### 3.3 预期回报计算（确定性公式，无需AI）

```python
def calc_expected_return(
    current_pe: float,
    target_pe_quantile: float,   # 默认40%分位
    earnings_growth_2yr: float,  # 净利润2年复利增速
    dividend_yield: float        # dv_ttm
) -> dict:
    
    earnings_contribution = (1 + earnings_growth_2yr) ** 2 - 1
    target_pe = get_historical_pe_quantile(target_pe_quantile)
    pe_change = (target_pe / current_pe - 1)
    valuation_contribution = pe_change * (1 + earnings_growth_2yr)  # 交叉项
    dividend_contribution = dividend_yield * 2
    
    total = earnings_contribution + valuation_contribution + dividend_contribution
    return {
        "earnings": earnings_contribution,
        "valuation": valuation_contribution, 
        "dividend": dividend_contribution,
        "total": total
    }
```

### 4.2 LLM调用架构

```
数据上下文构建
    → 主分析Prompt（DeepSeek-V3，成本低）
    → 结构化JSON输出
    → 叙述润色（可选，Claude Sonnet）
    → 写入报告数据库
```

为何分两步：
- DeepSeek-V3 便宜，处理结构化分析
- Claude 用于最终报告的叙述质量把关（可选，按需调用）

## 八、关键决策点

### 数据成本
| 方案 | 月成本 | 适合阶段 |
|------|--------|---------|
| akshare + 巨潮爬虫 + tushare基础版 | ~¥0~100 | Phase 1~2 |
| tushare积分版（2000积分）| ~¥200 | Phase 2 |
| Wind（最完整）| ¥2000+ | Phase 3 SaaS盈利后 |

### LLM成本（每份报告）
- DeepSeek-V3：~¥0.02/份（主分析）
- Claude Sonnet（润色）：~¥0.5/份（可选）
- 建议：默认DeepSeek，高付费用户开启Claude

### 最难攻克的点
1. **公告PDF解析质量**：部分公告是扫描件，需OCR（推荐 `pdfplumber` + `paddleocr`）
2. **机构一致预期**：免费源极少，Phase 1用管理层指引代替，Phase 3接朝阳永续API
3. **大基金/汇金持仓变化**：需从ETF季报中解析，延迟约45天，非实时
